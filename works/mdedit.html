VSCode Extension // 09

Markdown
Interactive Editor

マークダウンファイルをプレビュー状態のまま直接編集できるWYSIWYGエディター VSCode拡張機能。ソースを意識せず、見たまま書ける編集体験を提供する。

demo / screenshot
overview
PROBLEM
VSCodeでマークダウンを書くには記法を意識しながら編集する必要があり、プレビューとエディタを行き来するのが手間
SOLUTION
プレビュー表示のまま直接編集できるWYSIWYGモードを実現。マークダウン記法を意識せず、見たままの状態で文書を編集できる
TARGET USER
VSCodeでドキュメント・README・メモを書くエンジニアや、マークダウン記法に不慣れなユーザー
PLATFORM
VSCode拡張機能（Windows / Mac / Linux 対応）。*.mdファイルのカスタムエディターとして動作
features
✏️
WYSIWYGマークダウン編集
プレビュー表示のまま直接テキストを編集できる。見出し・太字・リスト・コードブロックなどマークダウン記法をビジュアルで確認しながら書ける。
📊
Mermaid図のインラインレンダリング
コードブロックにmermaidを指定するとflowchart・sequenceDiagram・erDiagramなどをWYSIWYG上でそのまま図として表示。設計ドキュメントやREADMEを記法を意識せず編集できる。
💻
コードブロック シンタックスハイライト
コードブロックで言語を指定するとWYSIWYG上でもシンタックスハイライトが適用される。JavaScript・PHP・TypeScriptなど主要言語に対応。
🖼️
画像ドラッグ&ドロップ
画像ファイルをエディタにドロップするだけで、指定ディレクトリへの保存とマークダウン記法の自動挿入を行う。保存先・サイズ上限・ファイル形式・重複処理を設定から制御できる。
📄
テンプレート機能
よく使うマークダウン構造をテンプレートとして登録・呼び出し。デフォルトテンプレートの復元にも対応。
☑️
インタラクティブなチェックボックス・ラジオボタン
チェックボックスはプレビュー上でそのままクリックして操作できる。さらにマークダウン標準では存在しないラジオボタン構文を独自サポートし、選択肢が一つに絞られる投票・確認用リストを作成可能。打ち合わせ用メモや議事録として活用しやすい。
🔄
テキストモード（別タブ表示）
ボタンワンクリックで同ファイルを通常のマークダウンテキストエディタとして別タブで開く。WYSIWYGで書いた内容をソースで確認・修正したいときに使う。
tech stack

technical notes

VSCode Custom Editor APIを使い、*.mdファイルに対してWYSIWYGエディターをカスタムエディターとして登録。marked.jsでマークダウンをHTMLに変換し、contenteditable属性を使ってインライン編集を実現している。

編集内容の変更検知にはMutationObserverを使用。DOM変更をリアルタイムに検知し、マークダウン形式に変換してVSCodeのドキュメントに書き戻すことで、通常のテキストエディタと同じ保存フローを維持している。

画像のドラッグ&ドロップはWebView内のdropイベントで受け取り、拡張機能側にバイナリデータをメッセージパッシングで渡して保存。ファイル名重複時の処理（上書き・日付付加・確認）を設定から選択できる。
