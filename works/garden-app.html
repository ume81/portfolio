家庭菜園
管理アプリ

植えた日を起点に「いつ何をすべきか」が時系列で一目わかる栽培タイムラインアプリ。天気予報と連動してリアルタイムにアドバイスを更新し、完了チェックとプッシュ通知で「やり忘れ」を防ぐ。

demo / screenshot
overview
PROBLEM
家庭菜園初心者は「いつ水をやればいい？」「追肥はいつ？」がわからず失敗しがち。既存アプリは静的な情報提供のみで、実行管理や天気連動がない
SOLUTION
植えた日を入力するだけでタイムラインが自動生成。天気APIと連動してリアルタイムにアドバイスを更新。完了チェック＋プッシュ通知でやり忘れを防ぐ
TARGET USER
家庭菜園の初心者〜中級者。失敗経験があってもっとうまくやりたい層。ベランダのプランター栽培から本格的な家庭菜園まで対応
PLATFORM
Android（MVP）。Expo製のため将来的にiOS対応予定。バックエンドはXserverでLaravel APIを稼働
features
📅
栽培タイムライン
植えた日を起点に種まき・間引き・追肥・収穫など各フェーズをタイムライン形式で表示。「いまここ」マーカーで現在のフェーズを強調し、次にすべきことが一目でわかる。
🌤️
天気連動アドバイス
OpenWeatherMap APIで47都道府県の天気を毎朝バッチ取得。霜注意・水やり不要・高温対策など条件に応じたアドバイスをタイムラインに反映。プッシュ通知でもリアルタイムに配信。
✅
完了チェック・リマインダー
各フェーズに「完了」ボタン。タップで実施日が自動保存され予定日とのズレを表示。予定日の朝にプッシュ通知、2日経過しても未完了なら再通知でやり忘れを防ぐ。
📆
カレンダー一覧
登録中の全野菜の作業イベントをカレンダー上に色分け表示。「今週のやること」リストで複数野菜の作業を一元管理できる。
📍
地域別スケジュール
冷涼地・中間地・暖地の3区分でタイムラインを補正。同じ野菜でも北海道と九州では植え付け時期が異なるため、地域設定に応じて栽培スケジュールを自動調整。
🥦
野菜データベース
ミニトマト・きゅうり・なす・ピーマンなど15種以上の主要野菜に対応。各都道府県農業試験場・農水省・JAの公的データをもとに栽培フェーズ・水やり・施肥・収穫基準を構造化して管理。
tech stack
Expo SDK 52 / React Native TypeScript Laravel 12 / PHP 8.3 OpenWeatherMap API Expo Push Notifications MySQL 8.0 Laravel Sanctum Laravel Socialite（Google認証） Filament（管理画面） Stripe / Laravel Cashier EAS Build Xserver
technical notes

天気連動が差別化の核。OpenWeatherMap APIで47都道府県分の天気データを毎朝6時のバッチ処理（Cron）で一括取得しサーバーキャッシュに保存。ユーザーのリクエストはキャッシュを参照する設計でAPI消費量を抑えている。API障害時はキャッシュ有効期限を6時間→24時間に延長して前日データで継続提供するフォールバック設計も実装。

通知は2層構成。追肥・収穫など栽培スケジュールに基づく通知はExpo Notificationsでローカルにスケジューリング（オフラインでも動作）。霜注意・高温対策など天気由来のアラートはLaravel→Expo Push Serviceを経由したサーバープッシュで配信。

バックエンドはLaravel 12 + Filament管理画面。野菜マスターデータの追加・編集をFilamentのCRUD画面から管理できる設計にし、コーディング不要でデータ拡充が可能。将来のサブスク機能（¥480/月）はStripe + Laravel Cashierで対応予定。
