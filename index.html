development approach

要件定義 → 設計 → 実装 → テスト の各フェーズを小さな単位で反復しながら進める開発スタイルを全プロジェクトで一貫して実践しています。AI開発ツールを単なる補助ではなく開発フローの中核に据え、設計段階での多角的レビュー・コミット単位のコードレビュー・自動テストを組み込むことで、個人開発でもチーム開発に近い品質保証を実現しています。

PHASE 01
要件定義
「誰の何を解決するか」を言語化。AIとの対話でユースケースの抜け漏れを洗い出す。
PHASE 02
設計
フェーズを小分けにして設計→実装→確認を繰り返す。全体を一気に設計せず細かく検証。
PHASE 03
実装
設計書をコンテキストとして渡しながらAIと協業。commitごとにAIによるコードレビューを実施。
PHASE 04
テスト
Playwrightによる自動テストを実施後、手動テストで最終確認。二段階のテストで品質を担保。
🤖
AI活用開発
Claude Code を開発フロー全体に組み込み、設計書を起点にAIへ文脈を固定することで仕様ブレを防ぎ、一貫した品質を維持。
⚡
並列設計レビュー
設計フェーズでサブエージェントを5並列起動し、アーキテクチャ・実装難易度とリスク・保守性と運用・パフォーマンス・代替案とトレードオフの観点を同時レビュー。単一視点では見逃しやすい問題を多角的に検出。
🔍
継続的コードレビュー
Git commitのたびにGeminiによる自動レビューを実施。バグ・設計のズレ・可読性の問題を早期に検出し、コード品質を継続的に向上。
🧪
二層テスト
Playwrightによるブラウザ自動テストで主要フローを網羅したうえで、手動テストで最終確認。二段階のテストで納品物の品質を担保。
